MIGRACIÓN DE USUARIOS
--------------------------------------------------------------------------------

Los usuarios NO fueron migrados porque:
  1. La migración se enfocó en crear la estructura y módulos
  2. Los usuarios requieren migración específica debido a:
     - Contraseñas encriptadas (diferentes entre Odoo 18 y 19)
     - Relaciones con partners
     - Permisos y grupos de seguridad
     - Preferencias de usuario

Para migrar usuarios, se requiere:
  1. Exportar usuarios desde OmniERP (sin contraseñas)
  2. Crear usuarios en nueva BD con contraseñas temporales
  3. Migrar partners asociados
  4. Asignar grupos y permisos
  5. Notificar a usuarios para que cambien contraseñas

PRÓXIMOS PASOS PARA MIGRAR USUARIOS
--------------------------------------------------------------------------------

1. Identificar usuarios activos en OmniERP que deben migrarse
2. Crear script de migración de usuarios que:
   - Exporte usuarios desde origen (vía MCP o SQL)
   - Cree usuarios en destino con contraseñas temporales
   - Migre partners asociados
   - Asigne grupos de seguridad correspondientes
3. Notificar a usuarios sobre nueva contraseña temporal
4. Configurar política de cambio de contraseña obligatorio
